# ADR-0011 — Локальный стор настроек: панель как реальная точка ввода (Города, §3)

- **Статус:** принято
- **Дата:** 2026-07-24
- **Контекст задачи:** [SPEC-panel.md](../SPEC-panel.md) §3, [ADR-0008](0008-script-shell-separation.md),
  [ADR-0009](0009-local-first-storage.md), [ARCHITECTURE.md](../ARCHITECTURE.md), [ROADMAP.md](../ROADMAP.md)

## Контекст

Обратная связь владельца (2026-07-23): панель была read-only оболочкой — все вкладки заглушки,
а Дашборд ничего не позволял настроить. Приоритет пересмотрен: **панель должна быть рабочей
точкой ввода**, и модель настроек зафиксирована — **настройки живут локально в конфиг-файле
(или БД); панель лишь читает/пишет их**. Этот слайс отменяет ранее запланированный
`prompt-14` (мастер настройки, откладывается) и вместо этого делает первый функциональный
кусок: блок «Города» на Дашборде (SPEC §3) — добавление города, настройка WB/Ozon по каждому
городу с моделью «наследует общее / переопределяет локально» и тумблером вкл/откл.

## Решение

1. **Локальный стор городов** (`app/scripts/cities.py`, `settings.city_config_path`,
   по умолчанию `config/cities.json`) — плоский JSON, атомарная запись (temp+`os.replace`,
   как в `app/storage/local.py`). Никакого хранилища секретов — прокси-креды лежат в файле
   открытым текстом (как и весь локальный режим, ADR-0009), но **маскируются** в любом
   печатаемом/логируемом/HTML-выводе.

2. **Модель «общее + локальное переопределение».** `defaults` — общий профиль по площадке
   (WB/Ozon): `enabled`/`proxy`/`interval_min`. Каждый город — `mode: inherit | override` на
   WB и Ozon отдельно; `override` несёт свои `enabled`/`proxy`/`interval_min`. Эффективное
   значение = `defaults[mp]`, если `inherit`, иначе поля override. `enabled: false` **убирает**
   пару (город, площадка) из рабочего набора целиком — не просто «не замерять», а
   отсутствует в `pairs`/`cities`.

3. **`list_effective()`** — единственная точка резолва inherit/override + drop отключённых
   пар; `app/scripts/control_panel.py::run` строит `WorkSet` через неё вместо
   `parse_proxy_map(settings.proxy_map_json)` напрямую. `CitySettings` расширен
   `marketplace_proxies: dict[Marketplace, str | None]` — прокси теперь резолвится независимо
   для WB и Ozon одного города (разные учётки на разные площадки — реальный кейс).

4. **Seed / обратная совместимость.** Нет файла `city_config_path` ⇒ `cities.load()` один раз
   формирует конфиг из текущего стора `regions` (code/name/geo) + `proxy_map_json`/интервалов:
   WB всегда `inherit`/`override(enabled=true)` (никогда не отключается сидированием — это
   сохраняет прежнее поведение «WB всегда со всеми активными регионами»), Ozon получает
   `override(enabled=false)` для регионов без гео `ozon` (сохраняет «Ozon только с гео»).
   `proxy_map_json` остаётся рабочим как источник сида; не устаревает.

5. **Панель — тонкая оболочка (ADR-0008).** Новые роуты `POST /cities`,
   `POST /cities/{code}/{mp}`, `POST /cities/{code}/delete` только парсят форму и вызывают
   `app/scripts/cities.py`; вся логика (резолв, атомарная запись, seed) — в скрипте, который
   работает и headless (`python -m app.scripts.cities`, и `cli.py cities ...`). Пустое поле
   прокси при сабмите **не обнуляет** сохранённое значение (`keep_proxy_if_empty`). Удаление
   города — деактивация из конфига, история измерений не трогается (та же семантика, что и
   `is_active=false` у `regions`/`products`).

## Что осознанно не входит в этот слайс

- **Редактор общего профиля** (изменение `defaults` через UI) и **глобальный тумблер
  площадки** — общий профиль показан в блоке «Города» только read-only; следующий слайс.
- Куки, параметры подключения, редактор скриптов — отдельные слайсы (SPEC §4/§5/§6).
- Хранилище секретов/шифрование — вне охвата локального режима (ADR-0009 §5).

## Последствия

- `docs/SPEC-panel.md` §3: блок «Города» из плана становится фактическим поведением.
- `docs/ROADMAP.md`: Фаза 8.2 — готово; 8.3 (куки) остаётся следующей, `prompt-14` (мастер
  настройки) снят с очереди — панель важнее как точка ввода прямо сейчас.
- Любой код, читавший `settings.proxy_map_json` напрямую для резолва прокси по региону,
  должен переходить на `cities.list_effective()` — `proxy_map_json` остаётся только как
  seed-источник, не как рабочий путь.
