# ADR-0006 — Панель управления, поставка и секреты

- **Статус:** принято (значения по умолчанию; владелец может переопределить)
- **Дата:** 2026-07-22
- **Контекст задачи:** [SPEC-panel.md](../SPEC-panel.md), [TZ.md](../TZ.md)

## Контекст

Владелец расширил объём: продукт (кодовое имя **Prizma**) поставляется клиенту с
**панелью управления** и **автоустановщиком**. Это отменяет пункт «не строим UI» из
исходного `TZ.md`. Нужно зафиксировать технические решения, чтобы разработка не ждала.

## Решения

1. **Панель — локальная веб-панель.** Бэкенд — **FastAPI** (единый Python-стек с ядром
   сбора). Фронтенд — статические страницы в стиле бренд-бука «Вектор·OS» (тёмная тема,
   IBM Plex, плавающие пилюли, светофор статусов). Пользователь открывает
   `http://127.0.0.1:<порт>` в браузере. Одинаково работает локально и на сервере (на
   сервере — вход по паролю).

2. **Поставка — zip с автоустановщиком под обе ОС.** Windows — `.exe`/`.bat`; Linux —
   установочный скрипт (или Docker/systemd). Установщик ставит зависимости (Python,
   Playwright+Chromium, `curl_cffi`, драйвер Postgres), разворачивает скрипты и панель,
   создаёт команду запуска.

3. **Секреты (куки, пароли, креды прокси) — шифрование at-rest.** Не в открытом виде и не
   в репозитории. Ключ — из ОС-хранилища или пользовательской passphrase. `.env`
   gitignored.

4. **Прогрев кук на Linux-сервере (без графики).** По умолчанию — прогрев на локальной
   машине владельца + перенос кук в панель. Опционально — headful Chromium через Xvfb на
   сервере. Выбор реализации — на фазе прогрева кук.

5. **Канонические поля цены (черновой финал).** `price`, `price_no_card`, `price_card`,
   `currency`, `availability`, плюс `measured_at`, `status`. Точный список полей WB
   (в т.ч. «с кошельком») и Ozon уточняется при реализации коллекторов.

## Последствия

- В стек добавляются FastAPI (панель) и упаковщик установщика; в роадмап — фазы «панель»
  и «упаковка/установщик».
- Панель управляет: общими профилями WB/OZ, городами (наследование + локальное
  переопределение), прогревом кук, маппингом источника/приёмника, редактором скриптов,
  логами (см. `SPEC-panel.md`).

## Открытые вопросы (follow-up)

- Механизм доставки обновлений клиенту (обновление версии панели/скриптов).
- Права доступа при серверной установке (роли).
- Финальный список ценовых полей (снимается при коллекторах).
