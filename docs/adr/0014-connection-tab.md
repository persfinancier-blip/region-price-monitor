# ADR-0014 — Вкладка «Параметры подключения»: UI поверх I/O-бэкенда

- **Статус:** принято
- **Дата:** 2026-07-24
- **Контекст задачи:** [ADR-0010](0010-io-adapters.md) (бэкенд, который эта вкладка
  оборачивает), [ADR-0008](0008-script-shell-separation.md), [ADR-0009](0009-local-first-storage.md),
  [ADR-0011](0011-local-settings-store.md), [docs/SPEC-panel.md](../SPEC-panel.md) §5 и §7,
  [ARCHITECTURE.md](../ARCHITECTURE.md)

## Контекст

`prompt-13` (ADR-0010) построил движок настраиваемых source/sink-адаптеров (`app/io/`) и
таблицу маппинга полей, но точкой входа оставался только сам файл `config/io.json`,
редактируемый вручную. SPEC-panel §5 (Фаза 8.4) требует вкладку панели — четыре
вертикальные закладки (5.1 Источник / 5.2 Маппинг входа / 5.3 Приёмник / 5.4 Маппинг
выхода) с предпросмотром и валидацией — как единственную предполагаемую точку ввода этого
конфига для оператора.

## Решение

1. **`io.json` остаётся единственным стором** — вкладка ничего не хранит сама, только
   читает/пишет `settings.io_config_path` через новый тонкий скрипт.

2. **`app/scripts/connection.py`** (ADR-0008, стиль `app/scripts/cities.py`):
   - `load()`/`save(config)` — атомарный раунд-трип `io.json` (temp + `os.replace`).
   - `columns(endpoint, table=...)` — читает фактический заголовок источника/приёмника
     (первая строка csv, строка заголовка xlsx, колонки таблицы БД) для ещё не сохранённого
     endpoint; отсутствующий файл/таблица ⇒ пустой список, без падения (форма без данных —
     нормальное состояние, не ошибка).
   - `validate_source`/`validate_sink` — прогоняют `app/io/mapping.validate` +
     `validate_known_fields` для каждой секции маппинга против реального заголовка,
     возвращают список всех нарушений (не бросают исключение) — панель показывает их
     инлайн.
   - `preview_source` — строит source через новый публичный
     `app/io/factory.build_product_source` и возвращает первые N строк products/regions,
     замаппленных в канонические поля — работает и для **несохранённого** конфига из формы.
   - `mask_database_url`/`resolve_database_url` — маскирование пароля в строке подключения
     и правило «пустой пароль при сохранении = оставить прежний», по аналогии с
     `cities.py::mask_proxy`/`keep_proxy_if_empty`.

3. **Публичные обёртки фабрики** (`app/io/factory.py`): `build_product_source(endpoint)` /
   `build_result_sink(endpoint)` — те же приватные `_build_*`, но вызываемые напрямую с
   произвольным `EndpointConfig`, а не только через `make_*(settings)` после сохранения
   файла. Поведение адаптеров не меняется.

4. **Реальная вкладка** (`app/panel/app.py` + `templates/connection.html`, тонкая
   оболочка): `GET /tab/connection` отрисовывает четыре под-вкладки, предзаполненные из
   текущего `io.json` (маскированный `database_url`); `POST /connection/source` и
   `/connection/sink` сохраняют соответствующую секцию; `POST /connection/preview`
   валидирует и возвращает предпросмотр для **посланного, необязательно сохранённого**
   конфига — оператор видит результат до нажатия «Сохранить». Строки таблиц маппинга
   берутся из `app/io/base.py` (`PRODUCT_FIELDS`/`REGION_FIELDS`/`RESULT_FIELDS`,
   обязательные помечены `*`) — единственный источник словаря полей, список ценовых полей
   (SPEC §9.5) растёт без правки шаблона. `connection` убран из `_PLACEHOLDER_TABS`.

5. **CLI**: `region-price-monitor connection {show,validate,preview}` зеркалит скрипт для
   работы без браузера.

## Что осознанно не входит

- **Мастер настройки (setup wizard)** — следующий слайс поверх этой формы.
- **Изменения поведения I/O-адаптеров** — только публичные обёртки над существующими
  `_build_*`; `make_product_source`/`make_result_sink` не тронуты.
- **Хранилище секретов** — `database_url` остаётся в открытом виде в `io.json` (ADR-0009),
  маскируется только при отображении/логировании.
- Другие вкладки (редактор скриптов, логи), общий профиль по умолчанию — не входят.

## Обоснование

- Тот же паттерн «тонкая панель поверх скрипта поверх движка» (ADR-0008), что и вкладка
  «Куки» — маршруты не содержат бизнес-логики, только парсинг формы и вызов
  `app/scripts/connection.py`.
- Предпросмотр/валидация несохранённого конфига (через публичные `build_*`) даёт
  оператору обратную связь до записи файла — не нужно сохранять «вслепую», чтобы увидеть
  ошибку маппинга.
- Маскирование пароля и «пустое поле = не менять» — уже проверенный паттерн из панели
  городов (прокси), переиспользован без изобретения нового UX.

## Последствия

- `app/panel/app.py::_PLACEHOLDER_TABS` больше не содержит `connection`;
  `tests/test_panel_placeholders.py` обновлён.
- `docs/SPEC-panel.md` §5 помечается доставленным; `ARCHITECTURE.md`/`ROADMAP.md`
  обновлены (Фаза 8.4 доставлена, следующий шаг — мастер настройки / редактор
  общего профиля).
- Существующие тесты `app/io/*` и `test_io_factory.py` остаются зелёными без изменений —
  `build_*` обёртки чисто аддитивны.
