# ADR-0005 — Метод сбора (обновление): куки + curl_cffi для Ozon, requests для WB

- **Статус:** принято — **заменяет метод из [ADR-0002](0002-scraping-strategy.md)**
- **Дата:** 2026-07-22
- **Контекст задачи:** [TZ.md](../TZ.md); подтверждение — [HANDOVER.md](../HANDOVER.md) §3

## Контекст

ADR-0002 назначал основным механизмом **headless-браузер Playwright** для обоих
маркетплейсов, считая, что Ozon «практически не берётся голым HTTP». Расширенный спайк
2026-07-22 (эта сессия) показал более надёжный, дешёвый и быстрый путь — и одновременно,
что браузер как основной способа чтения плохо масштабируется и всё равно упирается в
капчу/репутацию IP.

## Решение

**WB — читаем обычным `requests` через региональный прокси** (эндпоинт `card.wb.ru`,
регион = параметр `dest`). Браузер не нужен. Подтверждено спайком (см. `spike-report.md`).

**Ozon — читаем цену обычным запросом через `curl_cffi` с маскировкой TLS под Chrome
(`impersonate="chrome"`) + прогретые куки.** Ключевые факты спайка:

- обычный `requests` даёт **403** — Ozon фингерпринтит TLS/HTTP2 (JA3); поэтому именно
  `curl_cffi`, а не `requests`;
- нужны **прогретые куки**, добытые один раз в настоящем браузере (пройден антибот/капча);
- **регион зашит в куке** — цена меняется при смене региональной куки с одного IP
  (десятки регионов без парка региональных прокси для чтения Ozon);
- запрос идёт с того же IP, где грелись куки.

Проверено на живых данных: товар 3129447770 → HTTP 200, цена 2682 ₽ / без карты 5900 ₽ /
с Ozon-картой 2414 ₽; смена региона меняет цену.

**Браузер (Playwright/Chromium) остаётся, но только как инструмент разового «прогрева
кук»** — решить капчу, сохранить `storage_state`, отдать куки в рантайм. В рутинных
замерах браузер не участвует.

## Обоснование

- Доказано end-to-end на обоих маркетплейсах без дорогого антибот-сервиса.
- На порядок дешевле по CPU/RAM и выше пропускная способность, чем браузер на каждый замер.
- Интерфейс `MarketplaceCollector` из ADR-0002 сохраняется — меняются реализации, не
  оркестрация.

## Последствия

- Playwright переезжает из ядра сбора в модуль «прогрев кук» (запускается редко).
- Появляется новая сущность — **хранилище кук по (город × площадка)** с отслеживанием
  протухания; при протухании — повторный прогрев.
- `curl_cffi` добавляется в зависимости.
- Роадмап корректируется: WB-коллектор — на `requests`; Ozon-коллектор — на
  `curl_cffi`+куки; добавляется компонент прогрева кук.

## Открытые вопросы (follow-up, не блокируют старт)

- **Какая именно кука несёт регион** — определить экспериментально (сравнить наборы кук
  двух регионов). До этого регион задаём полным набором региональных кук.
- **Срок жизни кук Ozon** — измерить (прогонять чтение по времени до первого 403).
- **Прогрев кук на Linux-сервере без графики** — см. [ADR-0006](0006-panel-and-delivery.md).
