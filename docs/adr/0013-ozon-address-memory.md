# ADR-0013 — Ozon: регион по запомненному адресу + авто-починка (ревизия ADR-0012)

- **Статус:** принято
- **Дата:** 2026-07-24
- **Контекст задачи:** [ADR-0012](0012-cookie-collection-ux.md) (**ревизуется частично**),
  [ADR-0005](0005-scraping-method-update.md), [ADR-0008](0008-script-shell-separation.md),
  [ADR-0009](0009-local-first-storage.md), [ADR-0011](0011-local-settings-store.md),
  [docs/SPEC-panel.md](../SPEC-panel.md) §4/§8, [ROADMAP.md](../ROADMAP.md)

## Контекст

ADR-0012 реализовал `_switch_ozon_city`, переключающий город Ozon повторным заходом на
`https://www.ozon.ru/?city=<название>`. Это оказалось **неверной моделью**: Ozon не берёт
регион из query-параметра `?city=` — регион зашит в **набор кук** (семейство `__Secure-*` /
`abt_data`), которая записывается только когда оператор **выбирает адрес доставки** в UI.
Свободное имя города нельзя надёжно смэппить в канонический адрес Ozon — это и было багом.

Решение владельца (2026-07-24): **не гадать маппинг — один раз захватить реальный выбор
оператора и запомнить его, дальше чинить автоматически.**

## Решение

1. **`CookieBundle.address_label: str | None`** (`app/cookies/base.py`) — запомненный адрес
   доставки Ozon для города. У WB (`_session`) всегда `None`. Старые bundle без поля читаются
   как `None` (обратная совместимость, добавочное поле).

2. **Замена `?city=` на capture-and-select** (`app/cookies/warm.py`):
   - **Guided capture** (первый запуск / город без запомненного адреса) — открывает Ozon,
     ждёт (поллингом, не блокируя на `input()`), пока оператор выберет/подтвердит адрес в
     видимом окне, читает выбранный адрес со страницы и куку — сохраняет bundle с
     `address_label`.
   - **Auto-select** (обновление протухшей куки) — с уже запомненным `address_label` ведёт
     Ozon к выбору этого адреса из адресной книги аккаунта (совпадение по тексту); при неудаче
     (адрес не найден, нужен логин) — `AddressCaptureError`, шаг помечается `skipped` вместо
     тихого сохранения неверного региона.
   - `_switch_ozon_city` (`?city=`-хак) удалён. Капча-детект/опрос, per-step таймаут,
     `CancelToken`, `ProgressReporter`, `launch_browser`-сид для тестов — не изменились.
   - DOM/API выбора адреса в адресной книге Ozon эмпирически неопределены — реализовано
     защитно (селекторы best-effort, любая неудача → `AddressCaptureError`, не падение всего
     обхода).

3. **Два явных потока** в `app/scripts/cookies.py`:
   - `collect(marketplace, force=False)` — guided: обходит города **без** запомненного адреса
     (или все — с `force=True`). WB не меняется (одна сессия).
   - `refresh(marketplace)` — auto: для городов, у которых кука **протухла** и **есть**
     запомненный адрес, авто-переизбирает его. Города без адреса `refresh` не трогает — не
     гадает маппинг.
   - `status()` показывает `address_label` рядом с health-вердиктом.
   - `set_manual`/`clear` не меняются; `set_manual` может также задать `address_label`.
   - CLI: `cookies collect <mp> [--force]`, новый `cookies refresh <mp>`.

4. **Панель «Куки»** (`app/panel/app.py` + `templates/cookies.html`, тонкая оболочка): у
   каждого города Ozon — запомненный адрес (или «адрес не задан»), health, кнопка «Обновить
   протухшие» (`POST /cookies/{mp}/refresh`, запускает `refresh` фоновой джобой, тот же
   прогресс-поллинг что и у collect). Кнопка «Авторизоваться и собрать» осталась главной —
   guided для городов без адреса. Ручная вставка куки может задать `address_label`.

## Что осознанно не входит

- **Никакого маппинга свободное-имя → канонический адрес Ozon** — это и был баг; вместо этого
  захватывается реальный выбор оператора.
- **Никакой headless/серверной авто-починки** — починка требует видимый локальный браузер;
  случай голого Linux-сервера остаётся открытым вопросом (SPEC §9.2). `warm_if_stale` внутри
  прогонов коллектора **не** превращается в тихий headless-реварм — авто-починка это явное
  действие в панели/CLI.
- Шифрование кук/секретов не добавляется — куки + адрес хранятся локально, гитигнорятся
  (ADR-0009).
- Коллекторы WB/Ozon, парсер, прокси-слой, storage/IO seam, схема стора городов — не менялись
  за пределами чтения.

## Последствия

- ADR-0012 частично устарел: пункт про `?city=`-переключение (раздел 2, «авто-переключает
  город доставки… запрос с `?city=`») больше не отражает реализацию — заменён этим ADR.
  Остальные решения ADR-0012 (симметричная кнопка, тонкий скрипт, панель-оболочка, видимый
  браузер, куки без шифрования) остаются в силе.
- `docs/SPEC-panel.md` §4: региональность Ozon-куки описывается как «запомненный адрес», а не
  маппинг по имени города.
- Любой будущий код, читающий `CookieBundle.address_label`, должен считать `None` валидным
  состоянием (WB-сессия, старые bundle, ещё не настроенные Ozon-города).
